Ruby 笔记

Basic Syntax of Ruby

Ⅰ 面向对象

 1. 面向对象程式设计的基本特征
  a. 封装：把对象的属和行为结合成一个独立的单位，并尽可能隐蔽内部细节
  b. 多态：一个方法名属于多个对象，不同对象的处理结果不一样
  c. 继承：是可以定义一个子类来继承父类的方法
 2. 动态语言的设计理念
  a. 鸭子类型：关注的不是对象的类型本身，而是其使用方法
  b. 猴子补丁：可以重新打开类，重定义方法或添加新方法
 3. 返回值：任何表达方式都有返回值，表达式和语句没有区别
  a. 计算式的结果，赋值语句的值都是返回值
  b. 控制结构的最后一句的值是整个语句的返回值
 4. 代码规范
  a. 标点与排版
   i. 换行可以作为语句的结束标志，使用Unix风格换行符（\n）
    1) 可以使用“;”，一般用于大量重复简单语句，end前可以省略;
    2) 空类/空方法使用显式单行定义
   ii. 缩进与空白：两个空格（软Tab）
    1) 标点符号后一个空格，二元运算符左右各一个空格
    2) ./::/**运算符左右无空格
  b. 标识符命名规范：大小写敏感，不缩写，尽量不用非ASCII字符
   i. 变量/方法：小写单词，下划线连接
      方法的?/!结尾一般返回布尔值/修改自身
   ii. 常量（及类与模块）：大写字母开头单词，无下划线连接
  c. 类、方法与常量的文档标记
   i. 实例方法：类名#方法名
   ii. 其他：类/模块名::方法名、父类::子类、模块名::常量名


Ⅱ 控制语句

 1. 注释
  a. #：单行注释（到物理行行尾）
     = begin ~ = end：多行注释
  b. 特殊注释
   i. shebang注释：#! E:/ruby -w：置于首行，用于指定文件
   ii. 魔法注释：# encoding: UTF-8：置于次行（除非无shebang）可指定编码
   iii. 某些特定格式的注释可以生成API文档的HTML界面
 2. 条件语句（then皆可缺省，case与when缩进深度相同）
  a. if ~ then ~ elsif ~ then ~ else ~ end
  b. unless ~ then ~ else ~ end
  c. case ~ when ~ then ~ when ~ then ~ else ~ end
 3. 循环语句(do皆可缺省)
  a. for n in X do ~ end
  b. while ~ do ~ end
  c. until ~ do ~ end
 4. 异常处理语句
  a. begin ~ rescue (=> var) ~ (else ~)(ensure ~) end
   i. rescue：在begin部分出现异常时的执行部分
      当方法中无rescue时会逆向查找异常处理，跳出方法被上一层rescue捕捉
   ii. else：在begin部分未出现异常时的执行部分
   iii. ensure：后处理，无论处理是否正常进行都会执行的语句
   iv. retry：重新进行begin以下的处理
  b. 当在方法/类内或异常处理范围是整个程序时，可省略begin和end
 5. 控制语句：
  a. break语句
   i. 循环中跳出当前循环
   ii. 块中返回调用块的地方，计算结果会被忽略
      使用 break n 语句，可以指定break时的块返回值
  b. next语句
   i. 循环中跳过当前循环开始下一次循环
   ii. 块中使用next，会中断当前处理（数据清零返回nil）
      使用next n，可以指定next时的块返回值
  c. redo语句
   i. 循环中再执行一次相同的循环
   ii. 块中使用redo，可能会重复同一个参数陷入死循环
 6. 方法用语句
  a. 调用：除使用{}形式的块时，可以省略括号
   i. 参数传递顺序：普通参数→带默认值参数从前到后→*args
   ii. 用数列与散列传递参数：元素个数须与参数个数一致
    1) 用*arr将数组的各元素传递给方法
    2) 可以使用散列（键为符号）作为关键字传递参数（可以省略 {}）
  b. def语句：def method(a_1, a_2, …, a_n) ~ end：定义/重定义
     undef语句：undef :method/undef method：子类中删除超类方法
  c. return语句：可省略（没有返回值时默认为nil）
   i. 可用于立即跳出方法
   ii. return可返回多个值，用逗号连接
  d. yield(i_1, …, i_n; j_1, …, j_n)：相当于以参数i_n调用块函数
   i. 块变量数量不一致时，多余的无法传递，少的值为nil
   ii. 在块变量后面可以声明块局部变量（也可以在块中声明）
   iii. 块变量也可用*args传递多余的参数
 7. 类与模块用语句
  a. 创建类：class ~ def initialize ~ end ~ end
     创建模块：module ~ do_something end
  b. 创建类方法：
   i. Sin = Class.new
      class << Sin def ~ do_something end end
   ii. class ~ class << self def ~ do_something end end end
   iii. class ~.~ def ~ do_something end end
  c. 方法权限语句：单个设定/统一设定
   i. public（默认）：以实例方法的形式向外部公开该方法，一般可以缺省
   ii. private：在指定接收者的情况下不能调用该方法
    1) 只能在类或子类的实例方法中使用缺省self的方式调用该方法
    2) initialize 方法会被默认定义为private
    3) Object的顶层私有方法接收者是main对象（是Object类的实例）
   iii. protected：在同一个类中时可将该方法作为实例方法调用
  d. 类的继承
   i. class subclass < superclass ~ end
   ii. super(i_1, i_2, …, i_n)：沿用超类的方法
 8. 操作符：
  a. 修饰符：if/unless/while/until/rescue后置
   i. 修饰里修饰符最近的一条语句
   ii. 可以用()/begin ~ end合并多条语句为一条
   iii. 使用begin ~ end的循环时，循环体内的语句至少被执行一次
  b. 行继续符：置于物理行行尾的反斜杠，用于连接逻辑行
   i. 不是转义字符
   ii. 在""字符串中，""、''间连接、链式方法中可用
 9. 继承属性
  a. BEGIN{}与END{}：在其余所有代码之前/后执行
   i. 有多个BEGIN时，先出现的最先执行
   ii. 有多个END时，先出现的最后执行
   iii. 不可以用do ~ end代替{}
  b. __END__：单独一行表示程序结束，数据区开始
  c. proc.(x_1, x_2, ...)：相当于proc.call()，这个操作符不可重定义


Ⅲ 数据类型

 1. 编码：支持unicode
  a. 脚本编码：决定字面量字符串对象编码的信息，与脚本的字符编码一致
  b. 内部编码：从外部获取的数据在程序中如何处理的信息
  c. 外部编码：外部输出时与编码相关的信息
 2. 万物皆对象
  a. 对象：表现数据的基本单位，所有的对象都有标识（Object ID）和字面值
   i. 字面值：数字、文本字符、正则表达式
字面量 作用（括号内为10进制的值）
123 表示10进制整数
0123 表示8进制整数（83）
0o123 表示8进制整数（83）
0d123 表示10进制整数（123）
0x123 表示16进制整数（291）
0b1111011 表示2进制整数（123）
123.45 浮点小数
1.23E+04 浮点小数的指数表示法（1.23×10 的4次幂=12300.0）
1.23E-04 浮点小数的指数表示法（1.23×10 的-4次幂=0.000123）

   ii. 变量/常量（类/模块）通过储存对象的指向从而「引用」对象的字面值
   iii. 没有变量引用的对象会被GC回收
  b. Numeric类与Symbol类本身不可改变
   i. 这两种类没有显式声明的New方法
   ii. 方法输出结果事实上是引用的改变
 3. 实例的隐式创建
  a. 数字：数字不可简写，但可以加入下划线或其他识别符号
     两个特殊值：Float类中的Inifity和NaN
  b. 用""、''、//表示文本或文本模式
   i. ''只识别转义字符\'、\\，""识别所有转移字符以及行继续符
   ii. 转义字符
转义字符 意义
\x 等价于x本身，x是除下述字符外的所有字符
\a BEL字符，等价于\C-g或007
\b 退格符，等价于\C-h或010
\e ESC字符，等价于\033
\f 换页符，等价于\C-l或014

\s 空格符
\t 水平制表符，等价于\C-i或011
\v 垂直制表符，等价于\C-k或\013
\unnnn Unicode码点，不省略的十六进制数（0~10FFFF）

#{} 内插表达式
   iii. 正则表达式的模式匹配
    1) /^ABC$/：匹配行首/行尾
       /\AABC\z/:匹配字符串首/尾
       /ABC\Z/：当最后一个字符是\n时忽略而考察前一个字符
    2) /[ABC]/:A，B，C中的任何一个，与顺序无关
       /[A-Z]/：从A~Z中的任何一个
       /[A-Za-z_-]/：从-、_、A~Z、a~z中的任何一个
    3) /[^ABC]/：A，B，C以外的字符
       /[^A-Za-z]/：英文字母以外的字符
    4) /\p{Han}/：匹配汉字
    5) /A.C/：匹配AC，中间任意字符
    6) /\s\d\w\^\$\[/：匹配空白，任意数字，任意英文，^，$，[
    7) /A*BC/：匹配BC， ABC， AABC，……
       /A+BC/：匹配ABC， AABC，……
       /(AB)+c/：匹配ABC，ABABC，……
       以上两种匹配会尽可能多地匹配字符（贪婪匹配）
    8) /A*?BC/：匹配BC， ABC， AABC，……
       /A+?BC/：匹配ABC， AABC，……
       以上两种匹配会尽可能少地匹配字符（懒惰匹配）
       /A.+?B/ =~ "ABCDABCDABCD"
    9) /A?BC/：匹配BC， ABC
       /A{3}BC/：匹配AAABC
       /A{2, 4}BC/：匹配AABC，AAABC，AAAABC
    10) /(ABC|BCA)DEF/：在若干个候补中选择一个
  c. 用分界符+()/[]/{}/<>/||/!!/@@/%%/$$/--/__等创建文本
   i. %Q：双引号字符串，%q：单引号字符串
   ii. %w：字符串数组，%i：符号数组
   iii. %x：shell命令，相当于反引号
   iv. %r：正则表达式
  d. Here Document：<< HERE ~ (单独一行)HERE
   i. <<：分辨空格
      <<-：不分辨结束标志符的前导空白
      <<~：不分辨一切空白（便于缩进）
   ii. HERE可是任意标识符
    1) 前一个标识符使用'Here'可以不使用转义字符，且可以缩进
    2) 前一个标识符使用"Here"与Here一致，但标识符可使用空格
   iii. 可以用多个Here Document并与普通字符串混用
  e. shell命令：反引号``中的命令传递给 Kernel::`，返回结果字符串
     ?n：表示单字面量"n"，n可以为转义字符
  f. 表达式表示对象：
   i. [y_1, y_2]表示数组{x_1: y_1}/{x_2 => y_2}表示散列
    1) x_1为省略:的符号
    2) x_2与y可为数字，表达式，字符串，范围，符号
   ii. x..y/x...y表示一个Range类实例
   iii. :sym/:"sym"/:'sym'表示一个Symbol类实例

Ⅳ 方法与类

 1. 方法分类
  a. 实例方法：以特定对象种类为接收者的方法
  b. 类方法：以类为接收者的方法，可以用::运算符代替点运算符
     类方法包括class类的实例方法，也包括类对象的单例方法
  c. 函数方法：可以省略接收者的方法
 2. 类与模块
  a. 所有类都是Class类的对象
   i. BasicObject > Object > Module > Class
   ii. 模块是Module类的实例，自定义类是Class类的实例
   iii. 自定义类也是Object类的子类，自定义类的实例是Class类的实例的实例
  b. 模块不能拥有实例，不能被继承
  c. 模块提供的命名空间：不同模块的同名常量、方法被认为是不同的对象
   i. 未使用include时，使用模块名./::(方法/常量或方法)来调用
   ii. 调用模块方法时，依次在本方法，模块，超类的顺序寻找第一个该方法调用
 3. 运算符：大多都是实例方法的语法糖
  a. 不能重定义的运算符：
   i. =, ., ::, ?:
    1) ./::运算符形式（一般括号可以缺省）
     a) 方法可以用./::来调用
     b) 常量只能用::来调用
    2) 赋值运算符
     a) 「变量名 = 对象」：将对象赋值给变量
      i) 允许并行赋值、连环赋值
       (1) *操作符表示将数组拆分为多个元素
       (2) 左值用圆括号可以当作并行赋值
      ii) 对象部分为变量时，会传递变量所指对象的标识
     b) 二元运算符 op 都可以简写（伪操作符）
      i) var op= val 等价于 var = var op val
      ii) 读写器也可以用简写方式读写
   ii. 范围对象：.., ...：当两个参数为条件式或循环时会创建flip-flop布尔表达式
   iii. 内建语言：not, and, or, &&, ||
    1) 具有短路特性
    2) 在运算逻辑上用英文形式，在布尔计算上利用
  b. 其他运算符：
   i. defined?obj：返回是否被定义
   ii. !=/!~：会被自动定义为==/=~的反义
 4. 方法声明时的参数规则
  a. 省略参数：(x_1, x_2 = y_2, x_3 = y_3, ...)
   i. 指定默认参数的情况，所有带默认值的参数必须连续出现
   ii. method(*args)：将参数封装为数组
       (arg_1, *args, arg_2):最多出现一次，可以放在中间
  b. 关键字参数：(x_1, x_2: y_2, x_3: y_3, ...)
   i. 不同参数的位置可以改变，可以和普通参数同时使用
   ii. 定义不指定默认值的时候，y_n可以留空
   iii. 可以用**var来保存未定义的参数，以散列的形式保存
  c. 块作为参数的传递
   i. 当块作为Proc对象被传递时，作为普通参数
   ii. &block将显式传入的块封装为Proc对象，置于所有参数最后面
   iii. &block将符号转化为块：&method <=> Proc.new{|arg| arg.method}
 5. 单例方法
  a. 类方法是class类某个实例的单例方法
     单例类是其他类某个实例的单例方法，不能生成新的实例
  b. 类方法中画下划线的两个同时可以定义单例类
  c. Numeric类与Symbol类本身不看做对象引用，没有单例方法

 1. Ruby指令：ruby [option] [--] file [arguments]
  a. 命令行选项[options]
选项 意义
-0octal 用 8 进制指定 IO.gets 等识别的换行符
-a 指定为自动分割模式 ( 与 -n 或者 -p 选项一起使用时则将 $F 设为 $_.split($;)）
-c 只检查脚本的语法
-Cdirectory 在脚本执行前，先移动到 directory 目录下
-d、--debug 使用 debug 模式（将 $DEBUG 设为 true）
-e 'command' 通过 command 指定一行代码的程序。本选项可指定多个
-Eex[:in]、--encoding=ex[:in] 指定默认的外部编码（ex）以及默认内部编码（in）
-Fpattern 指定 String#split 方法使用的默认分隔符（$;）
-i[extension] 以替换形式编辑 ARGV 文件（指定 extension 时则会生成备份文件）
-Idirectory 指定追加到 $LOAD_PATH 的目录。本选项可指定多个
-l 删除 -n 或者 -p 选项中的 $_ 的换行符
-n 是脚本整体被 'while gets(); ... end' 包围（将 gets() 的结果设定到 $_ 中）
-p 在 -n 选项的基础上，在每个循环结束时输出 $_
-rlibrary 在执行脚本前通过 require 引用 library
-s 要使脚本解析标志（flag）的功能有效（'ruby -s script -abc'，则 $abc 为 true）
-S 从环境变量 PATH 开始搜索可执行的脚本
-Tlevel 指定不纯度检查模式
-U 将内部编码的默认值（Encoding.default_internal）设为 UTF-8
-v、--verbose 显示版本号，冗长模式设定为有效（$VERBOSE 设定为 true）
-w 冗长模式设定为有效
-Wlevel 指定冗长模式的级别 [0= 不输出警告，1= 只输出重要警告，2= 输出全部警告（默认值）]
-xdirectory 忽略执行脚本中 #!ruby 之前的内容
--copyright 显示版权信息
--enable-feature[, ...] 使 feature 有效
--disable=feature[, ...] 使 feature 无效
--external-encoding=encoding 指定默认的外部编码
--internal-encoding=encoding 指定默认的内部编码
--version 显示版本信息
--help 显示帮助信息
     --enable、--disable 选项可指定的功能名
功能名 意义
gems RubyGems 是否有效（默认有效）
rubyopt 是否引用环境变量RUBYOPT（默认引用）
all 上述功能是否全部有效

  c. < in.txt/> out.txt：定向输入/输出流
